# 界面功能

下面只是一个大体的界面功能，但是对页面上还没有划分。

## 登录注册

前端假数据先用 pinia 状态管理来表示登录状态。

## 员工端功能

### 今日课程

显示当日的需要上的课程的详细信息。

### 每周课程表

显示一个课程表，是一个二维表格，显示每周的周一至周五是否有课。行是周一至周日，列是时间数轴。从开始时间到结束时间对应的这个时间段，如果这个时间段有课，则这个区域显示为有背景颜色的显示课程名称和上课地点等信息的盒子，点击可查看详细信息。

员工可以切换不同周，查看当前周有哪些课。对于当日课程所在的列，背景颜色加深或是列头做特殊标识。

### 课程自评

对上完的课程，均可填写课程评价内容，主要聚焦于是否学懂，有没有不会的内容，之后我会接入 deepseek API 把内容发给他，然后 deepseek 会给出一个打分数字，从 0 到 100，分数越高，掌握度越高。这个分数就是员工自评分。

### 员工查看成绩

员工个人查看每节课得分情况，按“员工 × 课程安排”展示每一节课的详细信息和得分情况。也就是数据库里的员工每节课得分视图：v_employee_item_score

员工还可以看看自己在哪个类型上的课上薄弱。按“员工 + 课程类型”维度，汇总该员工在该系列类型的课程下所有上课记录的平均加权得分，也就是员工 × 课程类型综合得分视图：v_employee_course_score。这个可以做雷达图来可视化。雷达图的类型就是员工当前所处的培养计划内所有的课程的类型的种类，雷达图上的点就是员工在各个类型上的平均得分。

## 讲师端功能

### 今日主讲课程

显示当日的需要讲的课程的详细信息。

### 每周主讲课程表

和员工的课程表都类似，不过这个是讲师需要讲的课，所以显示的是讲师的课表。

### 讲师评分

既可以对员工的上课状态、回答问题情况等进行评语，交给 AI 来生成得分情况，从 0 到 100，也可以不进行评语，直接输入分数，无需让 AI 评价。

### 设置分数构成占比

讲师可以设置每一节课的讲师分数占比。而员工的自评分也可以根据这个占比来得出。例如讲师评分的占比就是 $80\%$，那么员工自评分占比就是 $20\%$，那么员工最终得分就是 $讲师评分 \times 80\% + 员工自评分 \times 20\%$。

### 讲师查看成绩

从“课程”维度汇总所有员工的表现的总体平均,分析课程整体效果。也就是课程整体评分视图：v_course_score。即看看老师所教的课的全班同学的所有分数的平均分。

讲师也可以细致地查看所教授的每一个员工在其所上的课程中的分数，按“员工 × 课程安排”展示员工在这个老师所教的每一节课的得分情况。每节课可以把每个员工的分数用柱状图或折线图可视化，来观察每节课的成绩分布。

## 培训大纲制定者端

### 创建培训计划

### 数据分析板块

培训大纲制定者可以查看培训计划内每个老师所看到的成绩，也就是每个老师的课程整体评分视图：v_course_score 和这个课的每个员工的单独分数分布等。

同时可以查看培训计划内培训计划整体评分视图：v_plan_score，也就是培训计划内所有课程的平均成绩，是综合该计划下所有课程、所有员工的表现，- 对每个培训计划做整体效果评估，做不同培训计划之间的对比分析。

界面上应该有比较直观的图表来展示这些数据。

### 培训计划板块

可以增删改查培训计划以及当前执行状态

### 培训计划内部添加课程板块

可以增删改查培训计划内的课程，以及课程安排例如时间地点等，一切符合数据库的设计即可。
