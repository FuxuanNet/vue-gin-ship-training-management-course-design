# 需求分析文档

总体要求

系统应支持员工培训计划的制定和执行，包括培训课程、培训时间、培训效果评估等，帮助员工提升技能水平。同时注意权限控制。
当前痛点/当前市场现状

- 培训计划的制定和管理混乱
- 员工难以掌握课程时间和安排
- 效果评估方式不规范，无法掌握员工技能掌握情况
市场现有解决方案/竞品分析
- 暂时还未调研
我们的解决方案
- 系统统一制定和管理各类培训计划
- 集中展示课程信息、时间、讲师等详细内容
- 通过问卷表单和实践考核，建立一个合理的评估机制
目标用户
- 船舶领域企业的管理岗领导与职工
产品用户类型
- 课程大纲制定者
- 讲师
- 员工
功能性需求
培训计划管理
权限：课程大纲制定者

1. 创建培训计划

- 输入：计划名称、开始日期、结束日期、目标员工范围
- 输出：计划编号、培训计划执行状态（规划中/进行中/已完成）
- 功能：检查上述信息是否合法，如果合法，就创建一个新的培训计划

2. 删除培训计划

- 输入：计划编号
- 功能：如果计划编号对应的培训计划存在，则删除

3. 修改培训计划

- 输入：计划编号
- 输出：修改后的培训计划
- 功能：如果计划编号对应的培训计划存在，则修改

4. 查询所有指定培训计划

- 输入：计划编号/计划名称
- 输出：指定培训计划

5. 查询所有计划执行状态

- 输入：计划执行状态
- 输出：所有满足条件的培训计划

6. 添加课程到培训计划中

- 输入：课程 ID、课程上课时间、课程上课地点
- 输出：课程在对应培训计划中的所处位置
- 功能：把对应的课程添加到培训计划中
课程管理

1. 创建培训课程

- 输入：课程名称、课程描述、课程要求、讲师名称
- 输出：课程编号

2. 查询、删除、修改培训课程

- 功能：对课程名称、课程描述、课程要求、讲师名称进行查询、删除与修改
时间安排

1. 提醒今日上课时间与地点以及课程要求
权限：讲师、员工

- 功能：类似于课程表，老师和员工都有
效果评估

1. 员工自评或提交问卷，形成掌握情况
权限：员工

- 输入：员工掌握情况问卷的填写内容
- 输出：掌握情况打分-员工
- 功能：用大语言模型对员工评价转换为掌握情况或对问卷填写情况建立合理的数学模型评价体系，得到员工在该课程的掌握情况打分

2. 讲师给员工进行评价，形成掌握情况
权限：讲师

- 输入：讲师给员工打的平时分
- 输出：掌握情况打分-讲师

3. 查看单个员工加权评价体系
权限：员工、讲师、课程大纲制定者

- 输入：掌握情况打分-员工 和 掌握情况打分-讲师
- 输出：加权评价评价分数
- 功能：是通过计算得到的

4. 查看课程加权评价体系
权限：讲师、课程大纲制定者

- 输入：对应课程的所有员工的加权评价体系
- 输出：该课程的加权评价分数
- 功能：是通过计算得到的

5. 查看培训计划加权评价体系
权限：课程大纲制定者

- 输入：对应所有课程的加权评价体系
- 输出：该培训计划的加权评价分数
- 功能：是通过计算得到的

界面需求

教师/员工端

- 设计清晰直观的课程表，简洁、便于查看，可以显示课程的上课地点、上课时间、上课要求等，可以支持在多系统查看，包括安卓、iOS、Windows、MacOS 等。
- 有对对应员工的分数及评价提交表单。

课程大纲制定者端

- 计划安排支持多时间线展示课程对应排布，清晰直观。
- 综合得分体系支持可视化分析看板，可视化员工的个人每节课得分，可视化课程针对所有员工的评分，可视化培训计划的评分统计，进而分析出员工在什么课程上的薄弱，课程本身建设有哪些薄弱，培训计划本身有哪些薄弱。
